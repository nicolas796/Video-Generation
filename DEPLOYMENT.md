# Render.com Deployment Guide

This guide will walk you through deploying the Product Video Generator to Render.com.

## Prerequisites

- A [Render.com](https://render.com) account
- Your API keys for:
  - [Pollo.ai](https://pollo.ai/) - Video generation
  - [ElevenLabs](https://elevenlabs.io/) - Voiceovers
  - [OpenAI](https://platform.openai.com/) - Script generation

## Quick Deploy

### Option 1: Deploy via Render Dashboard (Recommended)

1. **Push code to GitHub**
   ```bash
   git add .
   git commit -m "Prepare for Render deployment"
   git push origin main
   ```

2. **Create Web Service on Render**
   - Go to [Render Dashboard](https://dashboard.render.com)
   - Click "New +" → "Web Service"
   - Connect your GitHub repository
   - Render will auto-detect `render.yaml` and configure settings

3. **Add Environment Variables**
   
   In your Web Service dashboard, go to "Environment" and add:
   
   | Variable | Value | Required |
   |----------|-------|----------|
   | `POLLO_API_KEY` | Your Pollo.ai API key | ✅ |
   | `ELEVENLABS_API_KEY` | Your ElevenLabs API key | ✅ |
   | `OPENAI_API_KEY` | Your OpenAI API key | ✅ |
   | `POLLO_WEBHOOK_SECRET` | Random string (generate one) | ✅ |
   | `ADMIN_PASSWORD` | Secure admin password | ✅ |
   
   Note: `SECRET_KEY`, `DATABASE_URL`, and `APP_BASE_URL` are auto-generated.

4. **Deploy!**
   - Click "Manual Deploy" → "Deploy latest commit"
   - The build script will automatically:
     - Install dependencies
     - Run database migrations
     - Start the application

### Option 2: Deploy via render.yaml (Blueprint)

If Render detects `render.yaml`, it will suggest creating resources from the blueprint:

1. Push code with `render.yaml` to GitHub
2. In Render Dashboard, click "Blueprints" → "New Blueprint Instance"
3. Select your repository
4. Review and create resources

## Post-Deployment Configuration

### 1. Update Webhook URL in Pollo.ai

After deployment, update your Pollo.ai webhook URL:

```
https://your-app-name.onrender.com/webhook/pollo
```

### 2. Verify Database Migrations

The build script runs migrations automatically. To verify:
```bash
# In Render Shell (Dashboard → Shell)
flask db current
```

### 3. Access Admin Panel

- URL: `https://your-app-name.onrender.com/login`
- Username: `admin` (or your `ADMIN_USERNAME`)
- Password: Your `ADMIN_PASSWORD`

## Environment Variables Reference

### Required (Set in Render Dashboard)

| Variable | Description | Example |
|----------|-------------|---------|
| `POLLO_API_KEY` | Pollo.ai API key | `pollo_...` |
| `ELEVENLABS_API_KEY` | ElevenLabs API key | `sk_...` |
| `OPENAI_API_KEY` | OpenAI API key | `sk-proj-...` |
| `POLLO_WEBHOOK_SECRET` | Webhook verification secret | `random-string` |
| `ADMIN_PASSWORD` | Admin user password | `secure-password` |

### Auto-Generated by Render

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `SECRET_KEY` | Flask session secret (auto-generated) |
| `APP_BASE_URL` | Your app's URL |
| `FLASK_ENV` | Set to `production` |
| `PORT` | Port to bind (set by Render) |

### Optional

| Variable | Description | Default |
|----------|-------------|---------|
| `ADMIN_USERNAME` | Admin username | `admin` |
| `DEFAULT_VOICE_ID` | ElevenLabs voice ID | `XB0fDUnXU5powFXDhCwa` |

## Troubleshooting

### Database Connection Issues

Check the connection string format:
```bash
# In Render Shell
echo $DATABASE_URL
# Should start with postgresql:// (not postgres://)
```

The `config.py` handles the `postgres://` → `postgresql://` conversion automatically.

### Static Files Not Loading

WhiteNoise is configured in `app/__init__.py` for production. Check:
- Files exist in `static/` directory
- URLs use `/static/` prefix

### Health Check Failures

The health endpoint is at `/health`:
```bash
curl https://your-app-name.onrender.com/health
```

Should return:
```json
{"status": "healthy", "service": "product-video-generator", "timestamp": "..."}
```

### Build Failures

Check build logs in Render Dashboard:
- Ensure `build.sh` is executable: `chmod +x build.sh`
- Verify `requirements.txt` has all dependencies
- Check PostgreSQL connection during migrations

## Scaling & Production Considerations

### Upgrade from Free Tier

For production use, upgrade:
- **Web Service**: Starter ($7/month) or higher
- **Database**: Starter ($15/month) for persistence and backups

### File Uploads

Currently uploads are stored on ephemeral disk. For production:
- Files persist until service restarts
- Consider using AWS S3 for persistent storage
- Or upgrade to a plan with persistent disk

### Database Backups

On paid PostgreSQL plans, Render provides:
- Daily automated backups
- Point-in-time recovery
- Manual backup/restore options

## Updating the Application

Simply push to GitHub:
```bash
git push origin main
```

Render will auto-deploy if `autoDeploy: true` is set in `render.yaml`.

## Local Development

Continue using SQLite for local development:

```bash
# .env
FLASK_ENV=development
DATABASE_URL=sqlite:///app.db

# Run locally
python run.py
```
